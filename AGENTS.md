# Agent Guidelines for LarryVelez Personal Homepage

## Build & Test Commands
- **Local dev**: `bundle exec jekyll serve --livereload` (serves at http://localhost:4000)
- **Build only**: `bundle exec jekyll build` (outputs to `_site/`)
- **Install deps**: `bundle install`
- **No test suite** - this is a static Jekyll site

## Code Style & Conventions
- **HTML/Liquid**: Use Tailwind CSS classes via CDN, follow Jekyll conventions
- **Formatting**: 2-space indentation for HTML/YAML, 4-space for Ruby if needed
- **Naming**: Use lowercase with hyphens for files/folders (Jekyll convention)
- **Assets**: Store in `assets/` (css/, images/), reference via `{{ '/assets/...' | relative_url }}`
- **CMS Fields**: Use snake_case for front matter variables (e.g., `hero_headline`, `intro_text`)

## Important Notes
- **DO NOT edit `_site/`** - it's auto-generated by Jekyll
- **DO NOT manually edit `Czaruno/czaruno.github.io`** - automated deployment target
- **Content changes**: Edit `index.md` or use Decap CMS at `/admin/`
- **Deployment**: Auto-deploys to public repo on push to `main` via GitHub Actions
- **Baseurl**: Set to `""` for root deployment at czaruno.github.io
- **Local CMS**: Uses `local_backend: true` to bypass GitHub OAuth locally

## Architecture
Jekyll static site with Tailwind CSS (CDN), Decap CMS for content management, GitHub Actions for CI/CD to separate public repo.
